Getting Started

Set up and run the R-Type project offline.

Welcome

Welcome to the R-Type Project offline documentation! This guide will help you set up the project on your local machine and run it successfully. Follow the steps below to get started.

Prerequisites

Before you begin, make sure your environment meets the following requirements:

  • C++ Compiler (G++ or Clang)
  • CMake (minimum version 3.15)
  • Git for cloning the repository
  • SFML library (version 2.5.1 or higher)
  • vcpkg for dependency management

For detailed installation steps, follow the instructions in the repository's docs/setup-guide.html file.

Installation

Follow these steps to set up and build the project:

1. Clone the repository:

git clone https://github.com/n0v4org/Rtype.git
cd Rtype

2. Set up dependencies with vcpkg:

git clone https://github.com/microsoft/vcpkg.git
cd vcpkg
./bootstrap-vcpkg.sh
./vcpkg integrate install

Return to the R-Type repository root and install the required libraries.

3. Build the project:

./utils/build.sh

Running the Game

1. Run the Server

The server binary is named r-type_server. Use the following options:

./build/r-type_server -h        # Show help mode
./build/r-type_server -v        # Verbose mode
./build/r-type_server -gp 50000 # Run game server on port 50000 (default)
./build/r-type_server -lp 50002 # Run lobby server on port 50002 (default)

2. Run the Client

The client binary is named r-type_client. Use the following options:

./build/r-type_client -h                   # Show help mode
./build/r-type_client -ip <server_ip>      # Specify server IP
./build/r-type_client -sp 50000            # Specify server port (default 50000)
./build/r-type_client -cp 50001            # Specify client port (default 50001)
./build/r-type_client -lsp 50003           # Specify lobby server port (default 50003)
./build/r-type_client -lcp 50004           # Specify lobby client port (default 50004)

Troubleshooting

CMake version error:

cmake --version

Ensure CMake is version 3.15 or higher. Update it if necessary.

vcpkg or SFML not found:

./vcpkg integrate install
./vcpkg install sfml

Make sure vcpkg is integrated and the SFML library is installed.

Compilation issues:

g++ --version

Ensure your compiler supports C++17 or higher.
